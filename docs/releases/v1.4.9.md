# Release v1.4.9

## Summary

- Tightens the public Rust API around canonical entrypoints and guided
  construction so embedders have one documented path for builders,
  constructors, and runtime setup.
- Centralizes host assembly and runtime planning so startup, rebuild,
  plugin activation, rendering, and REPL coordination stop re-deriving
  the same policy in multiple places.
- Rebuilds intro/help rendering around semantic `osp` template blocks,
  ordered guide sections, shared ruled chrome, and explicit palette
  contracts so REPL startup and help output keep the intended shape
  across rich, compact, ASCII, and JSON surfaces.
- Fixes quick-search and structural DSL restore semantics so filtered
  help and intro payloads keep only the intended branches without
  resurrecting stale canonical buckets or leaking unrelated siblings.
- Consolidates architecture, contract, integration, end-to-end, and
  coverage-gate test suites so failures localize faster while duplicate
  happy-path coverage drops below the old churn-heavy baseline.

## Highlights

- Guided construction is now the first-class public API pattern:
  canonical builder and constructor entrypoints replace stale
  compatibility shims, and the crate docs explain when to use direct DTO
  values versus staged runtime builders.
- Startup, rebuild, and session initialization now share one internal
  host-assembly path, which removes repeated config, theme, plugin, and
  render-policy decisions from multiple coordinators.
- UI lowering now flows through a semantic render plan instead of mixing
  raw settings with partially resolved knobs across formatting and
  presentation layers.
- REPL intro/help content now round-trips through semantic guide data
  instead of markdown-like prose reconstruction, which makes section
  ordering, chrome, quick filtering, and output-format rendering
  deterministic.
- Quick search and addressed DSL restore now agree on a tighter contract
  for object fields, array elements, and canonical guide sections,
  closing the regressions that were keeping extra help/intro content
  alive after filtering.
- The test suite is reorganized around architecture, contract, and
  end-to-end boundaries, with new flagship intro contract coverage and a
  stricter release gate that now catches quick/restore drift, palette
  drift, and PTY-visible intro regressions before release.
