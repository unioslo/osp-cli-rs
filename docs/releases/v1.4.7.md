# Release v1.4.7

## Summary

- Adds the upstream command-policy mechanism needed for downstream products such as `osp-cli-uio` to consume generic auth and authorization plumbing cleanly.
- Carries generic auth metadata through plugin discovery, command listings, REPL surfaces, and dispatch without baking UiO semantics into the upstream crate.
- Keeps the release within the repo’s coverage and release-discipline constraints so `1.4.7` is ready to publish as the next consumable patch.


## Highlights

- New upstream command-policy types cover command paths, visibility, runnability, policy registration, and evaluated access results.
- Plugin `--describe` metadata can now declare generic auth requirements such as visibility mode, required capabilities, and feature flags.
- Plugin listings and REPL/help summaries now show compact auth hints derived from the same metadata path used for enforcement.
- Upstream auth documentation now explains the generic host/plugin contract and the boundary between upstream mechanism and downstream auth facts.
