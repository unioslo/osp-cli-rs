# Release v1.4.8

## Summary

- Adds a generic upstream native-command registry so downstream products can contribute top-level commands without forking the parser or special-casing CLI-only dispatch.
- Removes the stale legacy `workspace/` mirror so the repo and release tooling now match the actual single-crate package layout.
- Lands the first clean release path for the native-command seam by stabilizing clippy, test, and coverage behavior under `just release-check`.
- Relaxes the changed-file coverage floor to `85%` so the repo keeps a quality gate without forcing low-value coverage churn on every refactor.

## Highlights

- New upstream native command types and registry let top-level commands participate in help, completion, REPL surfaces, command policy, and dispatch through one shared metadata path.
- `just release-check`, `just verify-full`, and `just bump` no longer assume a compatibility workspace that has already been retired from the repo.
- The release path now survives full-suite `jq` and plugin fixture runs without brittle exact-string or environment-cascade failures.
- Local version bumps no longer depend on probing remote tags, which makes `just bump` usable even when SSH access or remote connectivity is unavailable.
