# Release v1.4.6

## Summary

- Promotes the root package to the real single-crate source of truth and removes the old mirrored `osp_*` module tree from the canonical build.
- Cleans up REPL and presentation behavior so `ui.presentation` is compiled into canonical config keys instead of being interpreted ad hoc throughout the UI.
- Finishes the first crates.io release path with trusted publishing plus the existing GitHub artifact workflow.


## Highlights

- Canonical root modules now live under `src/app`, `src/cli`, `src/config`, `src/core`, `src/dsl`, `src/plugin`, `src/repl`, `src/ui`, and `src/completion`.
- REPL intro behavior is now driven by `repl.intro` with presentation only seeding defaults, and verbosity can step the intro level up or down cleanly.
- Release automation now publishes to crates.io through the `crates-io-release` environment while still producing Linux, macOS, and Windows artifacts.


## Verification

- `./scripts/check-rust-fast.sh`
- `cargo test --all-features --locked`
- `cargo test --test contracts --all-features --locked`
- `cargo package`
- `cargo publish --dry-run`
