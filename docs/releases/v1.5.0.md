# Release v1.5.0

## Summary

- Tightens the public API around the real long-term seams for embedders and
  wrapper crates: typed config-store edit options, `App::builder()` as the
  canonical host front door, and wrapper-owned product defaults wired into
  normal runtime bootstrap.
- Removes retired DSL rollout compatibility surface, hardens runtime paths
  against panic-style failures, and adds regression guards that keep the
  non-test runtime free of `panic!`, `unwrap`, and `expect`.
- Rebuilds the rustdoc and repo-docs experience around credible entrypoints,
  wrapper guidance, and runnable examples, including a minimal product-wrapper
  example crate.

## Highlights

- `with_product_defaults(...)` is now a real host seam rather than a wrapper
  side channel. Native commands, `config get`, `config explain`, help, and
  rebuild flows all see the same product-owned defaults through normal runtime
  resolution.
- Explicit JSON output now stays machine-readable even in colored PTY sessions.
  JSON render paths force plain output instead of inheriting terminal chrome,
  and the release lane now covers both hermetic headless runs and explicit PTY
  JSON checks.
- Guided construction is cleaner: `App::builder()` is the canonical host entry,
  dead builder aliases are gone, `AppSession::builder()` remains as a neutral
  session seam, and the public surface is closer to one obvious path per job.
- The old `dsl2` / rollout compatibility layer is removed instead of being
  documented forever. The canonical DSL entrypoints are now the only public
  story.
- Public rustdoc is materially stronger: item-level contracts, feature-gate
  notes, runnable examples, and warning-free rustdoc checks are now enforced by
  scripts and tests instead of relying on memory.
- Repo docs are finally aligned with the code. The wrapper path now has a real
  tutorial plus a copyable example crate, and the stale LDAP guide was removed
  from this repository’s public docs surface.
