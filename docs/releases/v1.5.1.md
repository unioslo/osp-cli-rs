# Release v1.5.1

## Summary

- Stabilizes the internal host, UI, config, and REPL architecture after the
  `1.5.0` release without changing the public feature surface.
- Tightens rustdoc, release metadata, and local verification gates so examples,
  changelog state, and release readiness stay aligned with the crate API.
- Removes low-signal duplicate tests, restores app-host changed-file coverage,
  and keeps the patch release green under the full test and coverage lanes.

## Highlights

- The host/orchestration layer was cleaned up further around clearer ownership
  seams in app, config, UI, plugin, and REPL code. This is mostly maintenance
  work, but it reduces policy drift and makes later changes easier to localize.
- Public docs and rustdoc contracts are stricter. The crate now does a better
  job of documenting real entrypoints and release/readiness checks are less
  likely to drift from the code.
- The test suite is leaner where it was overlapping across unit, integration,
  and contract boundaries. The result is less duplicate promise coverage while
  keeping the release-path gates green.
